    BCC Connect

    How to use this workspace

    Short walk-through of every module — what each one does, how the data flows between them, and the one or two non-obvious things to know. Updated as the app grows.

    Getting started

    Five things to do the first time you sign in.

    1. Sign in with your @bluecollarcoach.us Microsoft 365 account. The whole workspace is gated by your tenant.
    2. Open Admin → Users & Roles and promote teammates you want to manage the workspace. The first signed-in admin is bootstrapped automatically.
    3. Visit Admin → Customer Types and shape the dropdown that powers CRM and Engagements (default: Coaching, Bookkeeping, Consulting, Marketing, Training, Technology, Other).
    4. Drop in connector credentials at Admin → Integrations for any external service you use (QBO, Google Ads, Meta, LinkedIn, Mailchimp, WordPress, GoDaddy, Slack, Teams). Click Test connection on each.
    5. Set your company info at Admin → Company Info — name, phone, email, address, tagline. This drives the letterhead on the rate sheet and other customer-facing artifacts.
    Tip: every action you take is written to the Activity Log (password-gated). Rotate the password from Admin → Security after first deploy.

    My Day

    Your personal landing each morning.

    Greeting + today's date + a row of KPI chips (sessions today, sessions tomorrow, total contacts). On the right: a Clock in / Clock out timer that persists across page reloads. Three cards underneath show today's sessions, tomorrow's sessions, and quick links to the things you start most often.

    At the bottom is your daily log: Wins, Blockers, Tomorrow's plan. Save once a day; the most recent saved time shows next to the heading.

    Sessions

    Coaching calendar, focused on the week ahead.

    Seven columns, one per day. Click a session card to edit; click + Add on a day to create one pre-filled to that date. Sessions can link to a CRM contact (dropdown) so the same person shows up across modules.

    Local-only sessions get a gold left border. Sessions synced to Microsoft 365 (when Graph sync ships) get a blue one.

    Cross-references: Sessions you add here automatically appear on My Day's today/tomorrow lists and the Scheduler month/year views. Same data, different lens.

    Scheduler

    Month and year overview.

    Toggle between month grid (clickable days that open a per-day modal with every session + event) and a 12-month year view (each month is a mini calendar; days with events are gold-tinted). Click any mini month to drill in.

    Read-only view — to add a session, the toolbar's + Session button jumps to Sessions with the chosen day pre-selected.

    CRM

    Contacts, companies, and the conversation history with each.

    List view

    Search by name, email, company, or tag. Filter by customer type (admin-driven), stage (lead/qualified/customer/inactive), or region (auto-populated from contact state codes). Click + New contact to add inline; click any row to open the detail view.

    Detail view

    Three sections side-by-side:

    • Profile: every field is inline-editable. Click Save changes to persist. Stage and customer type save on change (no button needed).
    • Needs: a checkable list. Strike through when met. Add inline; delete inline.
    • Conversations: a timeline of logged interactions (Note / Call / Email / Session / Meeting). Type a one-line summary + optional details; the entry is timestamped and attributed to whoever logged it.
    Print: the detail page has a clean print mode — hits Ctrl/Cmd+P to produce a contact handoff sheet for a colleague.

    Engagements

    Kanban pipeline of coaching/consulting deals.

    Six columns: New → Scheduled → Active → Waiting → Invoiced → Done. Drag a card between columns to move stage; each drag fires an audit row so you can see who moved what when.

    Each card: title, linked contact name, value, renewal/close date. Click to open the modal and edit. Service type is pulled from the Admin → Customer Types list so the same vocabulary works across CRM and Engagements.

    Rate Sheet

    Your coaching packages as a printable artifact + e-signature capture.

    The page renders a real letterhead with your company info (set on Admin → Company Info). Each package is edited inline — name, price, cadence (monthly / quarterly / annual / one-time), blurb. Add a package with the dashed "+ Add a package" button.

    The signature block at the bottom lets a customer draw on a canvas (mouse or finger). Save snapshots their name + linked contact + the current package list + the signature PNG into a bcc-rate-signature-* doc with timestamp + user-agent.

    Print: Ctrl/Cmd+P drops the toolbar/controls and gives you a clean letterhead for handoff. The signature block prints too, so a signed contract is one page.

    Documents

    Contracts, SOPs, customer files. Backed by Azure Blob.

    Drag-and-drop a file (or click to pick). Optionally set folder + tags. Up to 25 MB per upload. Files go to a Blob container scoped to your tenant; metadata stays in Cosmos so you can search by name, folder, or tag without downloading everything.

    Downloads happen via short-lived (15 min) signed URLs — no public link ever leaves the tenant.

    Marketing

    Track campaigns across Google Ads, Meta, LinkedIn, Mailchimp, WordPress, GoDaddy.

    The strip at the top mirrors your Admin → Integrations status per channel. Green dot = creds saved + last test succeeded. Configure first, then come here.

    Each campaign: name, channel, status (draft/active/paused/completed), budget + spend in USD, start/end dates, impressions/clicks/leads, free-form notes. Search + filter by channel or status.

    Bookkeeping

    Monthly P&L tied to QuickBooks Online.

    KPI strip at top: last-month revenue, expenses, net, YTD net — each with ↑/↓ delta vs the prior period. Underneath: a per-month table with net color-coded (gold = positive, burgundy = negative).

    Sync from QBO is enabled once the QBO connector in Admin → Integrations goes green. Until then, use + Manual entry to record a period by hand.

    Training

    Courses + lessons your team or clients work through.

    Each course = title + blurb + ordered list of lessons (title, video URL, duration minutes). Click any course card to edit; rows in the lesson editor have inline "+ Add lesson" / × remove.

    Enrollment + per-lesson progress tracking lives in bcc-enrollment-* docs (one per user × course); the dedicated learner view is on the roadmap.

    Events

    Workshops, meetups, intensives, with attendee bookings.

    Each event card shows when, title, location, and "N / capacity booked" + Open/Closed badge. Click to open the modal: title, start/end datetime, location, capacity, registration open Y/N, description.

    Inside the modal, a Bookings section lets you pick a contact from the dropdown (anyone in your CRM) or paste a free-form name. Each booking is timestamped; × to remove.

    Knowledge Base

    Internal playbooks in Markdown.

    Left pane: searchable article list with tag chips. Right pane: reader. Click + New article to draft; click Edit on any article to revise.

    Markdown supported lightly: # heading, ## sub, **bold**, `code`. Tag filter auto-populates from data.

    Admin Center

    Six tabs, in order of how often you'll touch them.

    Users & Roles

    Microsoft 365 tenant members pulled via Graph. Promote teammates to admin / mark inactive. The signed-in user is always auto-listed.

    Customer Types

    The dropdown source CRM + Engagements read. Add / rename / recolor / delete — saved as a single bcc-customer-types-v1 doc.

    Integrations

    Twelve connector cards. Paste credentials, click Test connection. Once a connector is green, the corresponding module (Bookkeeping for QBO, Marketing for ad channels, etc.) wires the saved creds in automatically — no env-var trip required.

    Company Info

    Legal name, display name, phones, email, website, address, tagline. Drives the letterhead on customer-facing artifacts (rate sheet today; contracts + reports next).

    Security

    Rotate the activity-log password + edit the domain allow-list. The tenant-level Entra restriction is already enforced; this is just a belt-and-suspenders layer for guest scenarios.

    Data & Backups

    Storage tally by key prefix + a one-click JSON snapshot export of every bcc-* doc in your browser. Useful before any architectural change.

    How data flows

    A two-sentence model that explains how every page stays in sync.

    Every save in any module writes a bcc-<thing>-<id> key to your browser's localStorage. A hook in bcc-api.js sees the write, debounces 1.2 seconds, and PUTs the doc to /api/data, which upserts into Cosmos under your tenant partition. Reads work the other way: on page load, bcc-api.js mirrors every prefixed doc from Cosmos into localStorage, then every page just reads from localStorage. Offline works. Saves queue and retry every 5 seconds + immediately when the browser comes back online.

    For the schemaless JSON shape of each module's docs, see docs/data-shapes.md in the repo.

    Activity Log

    Who did what, when. Password-gated.

    Every save / delete / sign-in / integration test fires a structured audit row. The feed filters by category (Admin / CRM / Sessions / Documents / Rates / My Day / Integrations / System) and by user. Sortable by time; clickable to drill in.

    Default password: bcc-audit-2026. Rotate from Admin → Security.

    Keyboard + a11y

    • Every interactive element has a visible focus ring + 44 px touch target.
    • Skip-to-content link at the top of every page (tab to it from the very first focus).
    • Toasts are aria-live; the offline banner is announced on connection drop.
    • Drawer + topbar are reachable from keyboard alone (no mouse-only menus).
